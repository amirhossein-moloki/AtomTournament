# توضیح مشکل پروفایل کاربری هنگام ثبت In-Game ID

در زمان به‌روزرسانی پروفایل، اگر کاربر از قبل عکس پروفایل داشته و کلاینت در درخواست PATCH مقدار `profile_picture` را به‌صورت رشته (آدرس URL ذخیره‌شده) ارسال می‌کرد، سریالایزر مقدار را به‌عنوان فایل جدید تفسیر می‌کرد و اعتبارسنجی تصویر روی آن اعمال می‌شد. از آنجا که URL با فرمت آپلود فایل مطابقت ندارد، این اعتبارسنجی خطا تولید می‌کرد و مانع ثبت `in_game_ids` می‌شد.

راه‌حل این است که تنها زمانی تصویر را برای اعتبارسنجی فایل در نظر بگیریم که واقعاً فایل جدیدی آپلود شده باشد (نمونه‌ای از `UploadedFile`). اگر مقدار `profile_picture` فقط رشتهٔ URL فعلی باشد، آن را نادیده می‌گیریم تا کاربر بتواند شناسه‌های داخل بازی را بدون بروز خطای تصویر به‌روزرسانی کند.
